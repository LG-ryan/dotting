## 0.1 로고 & CI 규칙 (SSOT)

### 로고 방향: 조용한 워드마크 + 분리된 시그니처

```
[로고 기본]
DOTTING          ← 정교한 워드마크만, 점(●●●) 없이
                    자간/획/끝처리 정교하게

[시그니처 별도]
●●●              ← 로고에 항상 붙이지 않음
                    상황별로 의미있게 사용
```

**왜 분리?**
- 로고에 항상 붙이면 유치해질 위험
- ●●●를 "의미 있는 UX 자산"으로 활용 가능
- "조용한 럭셔리" = 과한 장식 금물

### ●●● 사용 규칙

**의미**: `듣기 → 엮기 → 완성` (프로세스 상징)

| 사용처 | 형태 | 용도 |
|--------|------|------|
| **로딩 애니메이션** | ●○○ → ●●○ → ●●● | 진행 상태 표현 |
| **섹션 구분자** | · · · (작게) | 텍스트 블록 사이 |
| **푸터 시그니처** | ●●● | 브랜드 마침표 |
| **책 표지 워터마크** | ●●● (작게) | 결과물 브랜딩 |
| **완성 상태** | ●●● (강조) | "책이 완성됨" 표현 |

**디자인 토큰**

```css
/* ●●● 사이즈 */
--dot-size-sm: 4px;     /* 섹션 구분자 */
--dot-size-md: 6px;     /* 푸터/워터마크 */
--dot-size-lg: 8px;     /* 로딩/완성 상태 */

/* ●●● 간격 */
--dot-gap: 4px;         /* 점 사이 간격 */

/* ●●● 색상 */
--dot-active: var(--dotting-warm-amber);   /* 채워진 점 */
--dot-inactive: var(--dotting-border);      /* 빈 점 */
```

**로딩 애니메이션 예시**

```
●○○  듣는 중... (대화 화면)
●●○  엮는 중... (편집 화면)  
●●●  완성됨! (프리뷰/책)
```

### 편집자 톤 예시 (상태별)

| 상황 | 기존 (기능 중심) | 개선 (편집자 톤) |
|------|-----------------|-----------------|
| 로딩 | "로딩 중..." | "다음 질문을 준비하고 있어요" |
| 저장 완료 | "저장되었습니다" | "이야기가 안전하게 저장됐어요" |
| 에러 | "오류가 발생했습니다" | "잠시 쉬어갈게요. 다시 시도해주세요" |
| 10개 달성 | "무료 질문 한도 도달" | "10개의 소중한 이야기가 모였어요" |
| 책 완성 | "PDF 생성 완료" | "책이 완성됐어요. 이제 선물할 준비가 됐어요" |

### 톤 가이드라인

```
DO (해야 할 것)
├── 짧고 단정하게 (1-2문장)
├── "~해요" 존칭 사용
├── 결과물(책) 중심으로 표현
├── 성취감 주는 표현 ("모였어요", "완성됐어요")
└── ●●● 시그니처로 분위기 (이모지 대신)

DON'T (하지 말 것)
├── 이모지 사용 (🎉❤️📖 등) ← 조용한 럭셔리 해침
├── 기능 설명 ("AI가 처리 중입니다")
├── 부정적 표현 ("~밖에 안 남았어요")
└── 기술 용어 ("서버", "데이터", "프로세스")
```

### ●●● 노출 규칙 (남발 방지)

| 위치 | 노출 | 이유 |
|------|------|------|
| **푸터** | ✅ 항상 | 브랜드 시그니처 |
| **로딩 상태** | ✅ ●○○ 애니메이션 | 진행 표현 |
| **완성 상태** | ✅ ●●● 정적 | 달성 표현 |
| **일반 본문** | ❌ 사용 금지 | 남발 방지 |
| **버튼 텍스트** | ❌ 사용 금지 | 깔끔함 유지 |

## 4. 컴포넌트 가이드

### 4.1 버튼

```
┌─────────────────────────────────────────────────────┐
│  [Primary CTA]     → Deep Navy, 큰 패딩, 둥근 모서리   │
│  📖 책으로 완성하기                                    │
│  → 가장 중요한 액션에만 사용                           │
├─────────────────────────────────────────────────────┤
│  [Secondary]       → 테두리만, 투명 배경              │
│  👀 미리보기                                          │
│  → 보조 액션                                          │
├─────────────────────────────────────────────────────┤
│  [Ghost]           → 텍스트만, 밑줄                   │
│  나중에 할게요                                        │
│  → 취소/닫기 등                                       │
└─────────────────────────────────────────────────────┘
```

**CTA 문구 일관성 (전체 통일)**

> **SSOT 원칙**: 버튼 문구 = "지금 하는 행동"을 정확히 표현

| 단계 | 상태 | 메인 CTA | 보조 CTA |
|------|------|----------|----------|
| 랜딩 | - | "지금 시작하기" | "어떻게 만들어지나요?" |
| 대화 중 | - | "전송" | "잠깐 쉬어갈래요" |
| **10개 달성 모달** | 미결제 | "📖 책으로 완성하기" | "나중에 할게요" |
| 프리뷰 | 미결제 | "결제하고 완성하기" | "다시 만들어보기" |
| 프리뷰 | 결제완료 | "PDF 다운로드" | "실물 책 주문하기" |
| 결제 모달 | - | "결제하기 (39,000원)" | "나중에 할게요" |

**문구 구분 기준**:
- "📖 책으로 완성하기" → 완성 단계 진입 (축하 모달)
- "결제하고 완성하기" → 결제 행동 유도 (프리뷰)
- "결제하기" → 결제 모달 내부

### 4.2 채팅 말풍선

> **SSOT 결정 (2026-01-11)**: 사용자 말풍선은 **밝은 배경 + 진한 글씨**로 확정.
> 이유: 긴 글 가독성, 디지털 약자 친화, Deep Navy는 CTA에 집중.

```
AI 질문 (왼쪽)
┌────────────────────────────────────────┐
│ 🤖                                      │
│ ┌──────────────────────────────────┐   │
│ │ 어린 시절 가장 기억에 남는        │   │  ← Soft Cream (#FFFBEB) 배경
│ │ 여름 풍경이 있으신가요?           │   │     Deep Navy 글씨
│ └──────────────────────────────────┘   │     둥근 모서리 (왼쪽 상단만 각진)
│                                        │
└────────────────────────────────────────┘

사용자 답변 (오른쪽)
┌────────────────────────────────────────┐
│                                   👤   │
│   ┌──────────────────────────────────┐ │
│   │ 할머니 댁 마당에서 수박 먹던    │ │  ← Warm Gold (#FEF3C7) 배경
│   │ 기억이 제일 선명해요...         │ │     Deep Navy 글씨
│   └──────────────────────────────────┘ │     둥근 모서리 (오른쪽 상단만 각진)
```

**색상 정리**:
| 요소 | 배경 | 글씨 |
|------|------|------|
| AI 질문 | Soft Cream `#FFFBEB` | Deep Navy `#1A365D` |
| 사용자 답변 | Warm Gold `#FEF3C7` | Deep Navy `#1A365D` |

## 9. 구현 우선순위

> **핵심 원칙**: Phase는 '화면 우선순위'가 아니라 '공통 레이어 우선순위'다.
> Phase 1에서 토큰/CTA/진행률/모달을 고치면 A/B/C가 **동시에** 개선된다.

### 레이어 구조

```
Layer 1: 토큰 (색/타이포/간격)
    ↓ 적용
Layer 2: 공통 컴포넌트 (버튼/배지/모달/카드/입력)
    ↓ 적용
Layer 3: 화면 레이아웃 (A: 랜딩, B: 대화, C: 프리뷰)
```

### Phase 1 (즉시) - Layer 1 + 2 핵심

| 항목 | 레이어 | 영향 화면 |
|------|--------|-----------|
| 색상 CSS 변수화 | L1 | A, B, C 전체 |
| CTA 버튼 컴포넌트 | L2 | A, B, C 전체 |
| 진행률 컴포넌트 | L2 | B, C |
| 축하 모달 | L2 | B |
| 말풍선 스타일 | L2 | B |

### Phase 2 (1주 내) - Layer 1 완성 + Layer 3 시작

| 항목 | 레이어 | 영향 화면 |
|------|--------|-----------|
| 타이포그래피 적용 | L1 | A, B, C 전체 |
| 랜딩 Hero 섹션 | L3 | A만 |
| 대화 헤더/레이아웃 | L3 | B만 |
| 프리뷰 CTA 박스 | L3 | C만 |

### Phase 3 (2주 내) - 마감

| 항목 | 레이어 | 영향 화면 |
|------|--------|-----------|
| 애니메이션 | L2 | A, B, C 전체 |
| 반응형 최적화 | L3 | A, B, C 전체 |
| 접근성 검수 | 전체 | A, B, C 전체 |
