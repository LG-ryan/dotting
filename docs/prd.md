## 1. 목표 및 배경 (Goals & Background)

### 1.1 비전 (Vision Statement)

> **"모든 이야기는 계속된다... (Every story continues...)"**

DOTTING은 AI 기반의 추억 기록 서비스로, 대화형 인터뷰를 통해 개인의 삶의 이야기를 수집하고, 이를 아름다운 책과 디지털 콘텐츠로 제작합니다.

### 1.2 프로젝트 목표 (Goals)

| ID | 목표 | 성공 지표 |
|----|------|----------|
| G1 | AI 대화형 인터뷰로 타이핑 부담 제거 | 음성 입력 비율 80%+ |
| G2 | 경쟁사 대비 2주 내 결과물 제공 | 평균 제작 기간 14일 |
| G3 | 한국어 특화 스토리텔링 제공 | 고객 만족도 4.5+/5.0 |
| G4 | 1인 창업으로 시작 가능한 구조 | 월 BEP 26건 달성 |
| G5 | B2C → B2B → B2G 확장 가능 | 연간 매출 3억+ (3년차) |

### 1.3 배경 (Background Context)

**문제 정의**:
- 한국의 2025년 초고령사회 진입으로 부모님/조부모님의 이야기 기록 수요 급증
- 해외 서비스 (StoryWorth, Remento) 한국 미진출 → 시장 공백
- 기존 서비스의 한계: 타이핑 부담, 1년 대기, 일반적 질문, 개인화 부족

**시장 기회**:
- 부모님 선물 시장 약 2.4조 원
- 정년퇴직자 연간 30만 명 (명확한 이벤트 시점)
- 환갑/칠순/팔순 연간 150만 건

**선행 조건**:
- 시장조사 완료: `market-research-report.md`
- 경쟁사 분석 완료: `competitor-analysis.md`

## 2. 브랜딩 전략 (Branding Strategy)

### 2.1 브랜드 네이밍

#### 최종 후보 비교

| 항목 | DOTTING (도팅) | TRIPLEDOT (트리플닷) |
|------|---------------|---------------------|
| **의미** | DOT + ING (점을 찍는 중) | 세 개의 점 (...) |
| **연상** | "..."로딩, 이야기 계속됨 | 말줄임표, 미완의 이야기 |
| **발음** | 한국어 자연스러움 ✅ | 조금 길지만 직관적 |
| **SEO** | 검색 유리 (고유) | 검색 가능 |
| **확장성** | 다양한 서브 브랜드 가능 | 제한적 |
| **로고** | 애니메이션 가능 (.→..→...) | 정적 |

#### 브랜드 추천: **DOTTING (도팅)**

**선택 이유**:
1. **동적인 느낌**: "-ING" 접미사로 진행형, 살아있는 이야기
2. **로고 확장성**: 점이 하나씩 찍히는 애니메이션 로고 가능
3. **서브 브랜드**: DOTTING Book, DOTTING Story, DOTTING Family 등
4. **한국어 발음**: "도팅" - 2음절, 기억하기 쉬움
5. **도메인**: dotting.kr, dotting.co.kr 확보 가능성

### 3.3 핵심 기술 구현 방안

#### 3.3.1 AI Interviewer Engine (핵심 차별화)

```python
# 개념적 아키텍처
class DOTTINGInterviewer:
    """
    DOTTING의 핵심 차별화 기술: 대화형 AI 인터뷰어
    """
    
    def __init__(self, user_id: str):
        self.context = ConversationContext(user_id)
        self.emotion_detector = EmotionDetector()
        self.question_generator = DynamicQuestionGenerator()
        
    async def conduct_interview(self, audio_input: bytes) -> InterviewResponse:
        # 1. 음성 → 텍스트
        transcript = await self.speech_to_text(audio_input)
        
        # 2. 감정 분석
        emotion = self.emotion_detector.analyze(transcript, audio_input)
        
        # 3. 맥락에 저장
        self.context.add_response(transcript, emotion)
        
        # 4. 후속 질문 생성 (핵심 차별화)
        next_question = await self.question_generator.generate(
            context=self.context,
            emotion=emotion,
            depth_level=self.context.current_depth
        )
        
        return InterviewResponse(
            processed_text=transcript,
            emotion=emotion,
            next_question=next_question,
            story_progress=self.context.progress
        )
```
