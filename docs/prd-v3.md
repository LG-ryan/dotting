# PART 1: 문제 정의 (명확화)

## 1.1 Job-to-be-done (JTBD)

> **"물어보고 싶은데, 어떻게 물어봐야 할지 모르겠다"**

### 고객이 해결하고 싶은 근본 문제

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   상황 (When)                                                   │
│   ─────────────────────────────────                             │
│   • 부모님이 연로해지셨을 때                                    │
│   • 특별한 이벤트 (정년퇴직, 환갑, 칠순) 가 다가올 때           │
│   • "우리 부모님 이야기를 기록해두고 싶다"는 생각이 들 때       │
│                                                                 │
│   동기 (I want to)                                              │
│   ─────────────────────────────────                             │
│   • 부모님의 인생 이야기를 체계적으로 기록하고 싶다             │
│   • 나도 몰랐던 부모님의 과거를 알고 싶다                       │
│   • 세상에 하나뿐인 의미 있는 선물을 드리고 싶다                │
│                                                                 │
│   기대 결과 (So that)                                           │
│   ─────────────────────────────────                             │
│   • 부모님이 살아계실 때 이야기를 남길 수 있다                  │
│   • 가족 모두가 소중히 간직할 수 있는 유산이 된다               │
│   • 부모님도 자신의 인생을 돌아보며 뿌듯해하신다                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 기존 방법이 안 되는 이유

| 방법 | 문제점 | 결과 |
|------|--------|------|
| **직접 물어보기** | "뭘 물어봐야 하지?" 막막함 | 시작도 못 함 |
| **녹음하기** | 정리가 안 됨, 들을 수 없는 파일만 쌓임 | 방치 |
| **영상 찍기** | 카메라 앞에서 어색, 편집 어려움 | 포기 |
| **글로 쓰기** | 타이핑 어려움 (시니어), 시간 부족 | 미완성 |
| **StoryWorth** | 52주 기다려야 함, 텍스트 입력 필요 | 중도 이탈 |

### DOTTING의 해결책

> **"AI가 물어보고, 당신은 편한 방식으로 답하면 된다"**

```
문제: 어떻게 물어봐야 할지 모르겠다
     → 해결: AI가 적절한 질문을 해드립니다

문제: 정리가 안 된다
     → 해결: AI가 자동으로 예쁘게 정리합니다

문제: 타이핑이 어렵다 (시니어)
     → 해결: 음성으로 말만 하세요

문제: 말하기가 어색하다 (젊은 세대)
     → 해결: 텍스트로 작성하세요

문제: 너무 오래 걸린다
     → 해결: 3분씩 조금씩, 또는 하루에 몰아서
```

## 1.2 핵심 가설 (검증 필요)

### 반드시 검증해야 할 가설들

| # | 가설 | 검증 방법 | 성공 기준 |
|---|------|----------|----------|
| H1 | 자녀가 세팅하면 부모님은 "말만" 할 수 있다 | 10가정 테스트 | 8/10 성공 |
| H2 | 시니어도 음성 인터뷰 UX를 이해할 수 있다 | 60대+ 5명 테스트 | 4/5 완료 |
| H3 | AI 질문이 자연스럽고 깊이 있다고 느낀다 | 인터뷰 후 설문 | 만족 4.0/5.0+ |
| H4 | 중간에 포기하지 않고 10개 이상 이야기 완료 | 완료율 측정 | 50%+ |
| H5 | 완성된 결과물에 감동한다 | 인터뷰 후 설문 | 추천 의향 80%+ |

### 가설이 틀렸을 때의 피벗

| 가설 | 틀렸을 때 대안 |
|------|--------------|
| H1 | 자녀가 옆에서 함께 진행하는 모드 추가 |
| H2 | 전화 통화 방식으로 전환 (웹 아닌 전화) |
| H3 | 사람 에디터 개입 (Human-in-the-loop) |
| H4 | 게임화 강화 또는 더 짧은 세션 |
| H5 | 편집 서비스 추가 (AI → 사람 검수) |

│      • 응답자: 부모님 (50~80대)                                 │
│      • 동기: 선물, 기록, 감사                                   │
│      • 트리거: 어버이날, 생신, 정년퇴직                         │

│                                                                 │

│                                                                 │
│   📈 확장 타겟 (Phase 2~)                                       │
│   ─────────────────────────────────                             │
│                                                                 │
│   B. 나 자신의 이야기 (자기 기록)                               │
│      • 구매자 = 응답자 (본인)                                   │
│      • 동기: 자기 성찰, 미래 자녀에게 전달                      │
│      • 트리거: 새해, 기념일, 인생 전환점                        │
│      • 입력: 텍스트 선호                                        │
│                                                                 │
│   C. 반려동물 이야기                                            │
│      • 구매자 = 응답자 (보호자)                                 │
│      • 동기: 함께한 시간 기록, 추모                             │
│      • 트리거: 반려동물 생일, 무지개다리                        │
│      • 입력: 텍스트                                             │
│      • 특징: 감정적 동기 매우 강함                              │
│                                                                 │
│   D. 떠난 이에 대한 이야기 (추모)                               │
│      • 구매자 = 응답자 (유족)                                   │
│      • 동기: 추억 정리, 기억 보존                               │
│      • 트리거: 기일, 추모                                       │
│      • 입력: 텍스트                                             │
│      • 특징: 민감함, 신중한 접근 필요                           │
│                                                                 │
│   E. 커플/친구/동료 이야기                                      │
│      • 구매자: 관계의 한쪽                                      │
│      • 응답자: 함께 또는 각자                                   │
│      • 동기: 우정/사랑 기념                                     │
│      • 트리거: 기념일, 결혼, 졸업                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

### 제품 구조: "누가 누구의 이야기를" 선택

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   프로젝트 생성 시 선택:                                        │
│                                                                 │
│   Q1. 누구의 이야기인가요?                                      │
│   ─────────────────────────────────                             │
│   ○ 부모님 / 조부모님                                          │
│   ○ 나 자신                                                    │
│   ○ 반려동물                                                   │
│   ○ 떠나보낸 분                                                │
│   ○ 연인 / 친구 / 동료                                         │
│   ○ 기타 (직접 입력)                                           │
│                                                                 │
│   Q2. 누가 이야기를 들려주나요?                                 │
│   ─────────────────────────────────                             │
│   ○ 본인이 직접                                                │
│   ○ 다른 사람 (링크 전송)                                      │
│   ○ 함께 (같이 진행)                                           │
│                                                                 │
│   → 선택에 따라 질문 세트, UX, 톤이 달라짐                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 입력 방식: 음성 vs 텍스트

| 방식 | 적합 대상 | 장점 | 단점 |
|------|----------|------|------|
| **음성** | 시니어, 타이핑 어려운 분 | 쉬움, 자연스러움 | 기술 복잡, 오류 가능 |
| **텍스트** | 젊은 세대, 신중한 성격 | 정확, 수정 용이 | 시간 소요 |
| **하이브리드** | 모든 사용자 | 선택권 | 개발 복잡 |

**결론**: **하이브리드 (사용자 선택)** 방식 채택
