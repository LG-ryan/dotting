## 1.1 최종 패키지 정의

### 패키지 개요

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   📚 Prologue - Essay - Story - Heritage                        │
│                                                                 │
│   문학적 서사 + 브랜드 철학 완벽 결합                           │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ○ ○ ○  Prologue (프롤로그)                                   │
│           이야기의 시작 - 무료 체험                             │
│           0원                                                   │
│                                                                 │

│   ● ● ○  Story (스토리) ⭐ EARLY BIRD                          │
```

## 1.2 네이밍 전략 근거

### 선정 이유

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   🏆 Prologue - Essay - Story - Heritage 선정 이유              │
│                                                                 │
│   ─────────────────────────────────                             │
│                                                                 │
│   1. Prologue (프롤로그) vs Preview                             │
│   ─────────────────────────────────                             │
│   ✅ Prologue 승리                                              │
│   • 감성적: "당신의 위대한 이야기가 시작된다"                   │
│   • 문학적: 책의 서문, 집필의 시작                              │
│   • Preview는 기능적 (OTT 느낌)                                 │
│                                                                 │
│   2. Essay (에세이) vs Chapter                                  │
│   ─────────────────────────────────                             │
│   ✅ Essay 승리                                                 │
│   • 완결성: 그 자체로 완성된 문학 장르                          │
│   • Chapter는 "부분" 느낌 (불완전)                              │
│   • 3~4만원대 실속형 선물 포지셔닝                              │
│                                                                 │
│   3. Story (스토리)                                             │
│   ─────────────────────────────────                             │
│   ✅ 가장 직관적                                                │
│   • 도팅의 본질: "이야기"                                       │
│   • 브랜드 연결: "도팅 스토리"                                  │
│   • 대중적 호소력                                               │
│                                                                 │
│   4. Heritage (헤리티지) vs Legacy                              │
│   ─────────────────────────────────                             │
│   ✅ Heritage 승리                                              │
│   • 명품 느낌: 백화점/럭스 채널 용어                            │
│   • 문화적 유산 (vs 법적 유산)                                  │
│   • 2025/2026 프리미엄 트렌드 부합                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 2.2 구현 세부사항

### 오디오 QR 코드 (Story, Heritage)

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   🎵 오디오 QR 코드                                             │
│                                                                 │
│   위치: 책 속지 또는 각 챕터 시작 부분                          │
│   내용: 부모님의 실제 목소리 녹음                               │
│   재생: QR 스캔 → 웹 플레이어                                   │
│                                                                 │
│   기술 스택:                                                    │
│   • 오디오 파일: messages.audio_url 활용                        │
│   • QR 생성: qrcode.js 또는 서버 사이드                         │
│   • 재생: /play/[audioId] 페이지                                │
│                                                                 │
│   비용:                                                         │
│   • QR 인쇄: 추가 비용 거의 없음                                │
│   • 스토리지: Supabase Storage (기존 인프라)                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 촉각적 차별화

```
┌─────────────────────────────────────────────────────────────────┐

│                                                                 │
│   구현:                                                         │
│   • 인쇄소 위탁 (자체 관리 X)                                   │
│   • 샘플 제작 후 품질 검증                                      │
│   • 비용 마진에 포함                                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
